# Vultron Log Replay Demo

## Overview

The Log Replay demo is a new visualization mode that reads JSONL case log files generated by container-based Vultron demos and displays them using the same visual swimlane interface as the interactive demos.

## What Was Built

### 1. **Demo Selector Enhancement** ([DemoSelector.tsx](src/DemoSelector.tsx))
   - Added a third demo option: "Log Replay"
   - Users can now toggle between:
     - Single Vendor (interactive)
     - Multi-Vendor (interactive)
     - **Log Replay (visualization only)**

### 2. **JSONL Parser** ([utils/jsonlParser.ts](src/utils/jsonlParser.ts))
   - Parses JSONL format case log files
   - Extracts `CaseLogEntry` objects with event metadata
   - Helper functions to:
     - Parse JSONL line-by-line
     - Extract actor types from URLs
     - Extract participant status information
     - Merge and sort entries from multiple log files by timestamp

### 3. **Log Event Mapper** ([utils/logEventMapper.ts](src/utils/logEventMapper.ts))
   - Converts `CaseLogEntry` events to `TimelineEvent` objects
   - Handles these event types:
     - `submit_report` → Creates decision node + consequence nodes
     - `engage_case` → Shows participant engagement
     - `add_participant_status` → Displays RM/VFD state changes
     - `remove_embargo_event_from_case` → Shows embargo termination
   - Automatically initializes participant state (Finder, Vendor, CaseActor)
   - Builds complete timeline with proper spacing and positioning

### 4. **Log Replay Component** ([App-logreplay.tsx](src/App-logreplay.tsx))
   - File upload interface for selecting JSONL files
   - Supports multiple files (upload all participant logs at once)
   - Renders using existing swimlane visualization components:
     - `ActorPanel` for participant lanes
     - `AnimatedNode` for timeline events
   - Features:
     - Synchronized scrolling between sidebar and timeline
     - Event log panel showing all activities
     - Hover effects on timeline nodes
     - Reset/reload functionality

## How to Use

### Step 1: Generate Log Files
Run a container-based demo to generate JSONL logs:

```bash
bash integration_tests/demo/run_multi_actor_integration_test.sh two-actor
```

This creates log files in:
- `devlogs/two-actor/finder/urn_uuid_*-case-log.jsonl`
- `devlogs/two-actor/vendor/urn_uuid_*-case-log.jsonl`
- `devlogs/two-actor/case-actor/urn_uuid_*-case-log.jsonl`

### Step 2: Open the UI Demo
```bash
cd ui
npm run dev
```

### Step 3: Load Logs
1. Click "Log Replay" in the top demo selector bar
2. Click "Choose Files" in the upload dialog
3. Select all three JSONL files from the `devlogs/two-actor/` folders
4. The timeline will automatically render

## Event Type Mapping

| CaseLogEntry eventType          | Visualization                                    |
|----------------------------------|--------------------------------------------------|
| `submit_report`                  | Decision node in Finder lane + consequence in Vendor |
| `engage_case`                    | Decision node showing case engagement            |
| `add_participant_status`         | Consequence node showing RM/VFD state update     |
| `remove_embargo_event_from_case` | Decision node showing embargo termination        |
| `demo_verification`              | Skipped (not visualized)                         |

## Supported Log Fields

From `CaseLogEntry`:
- `eventType` - Type of event
- `receivedAt` - Timestamp for ordering
- `payloadSnapshot.actor` - Who performed the action
- `payloadSnapshot.object` - Event details (status, embargo, etc.)
- `payloadSnapshot.object.rmState` - Report Management state
- `payloadSnapshot.object.vfdState` - Vendor Fix Development state
- `payloadSnapshot.object.caseStatus.emState` - Embargo Management state
- `payloadSnapshot.object.caseStatus.pxaState` - Public/eXploit/Attacks state

## Testing

To verify the implementation works:

1. Generate fresh logs:
   ```bash
   bash integration_tests/demo/run_multi_actor_integration_test.sh two-actor
   ```

2. Start the UI:
   ```bash
   cd ui && npm run dev
   ```

3. Navigate to the UI in your browser

4. Click "Log Replay" and upload the three JSONL files

5. Verify:
   - All three participants (Finder, Vendor, CaseActor) appear in lanes
   - Timeline events are visible and positioned correctly
   - Events are in chronological order
   - Hovering events works
   - Event log panel shows activity
   - Scrolling is synchronized

## Notes

- The implementation reuses existing visualization components for consistency
- No changes to the protocol or container demos were needed
- The parser is flexible and can handle additional event types by extending the mapper
- File upload was chosen over direct file system access for browser compatibility
