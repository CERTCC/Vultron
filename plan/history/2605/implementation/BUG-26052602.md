Symptom: 404 Not Found when verifying stored report in TestTriggerLogCommit,
TestWaitForFinderLogEntry, TestVerifyFinderReplicaState, TestRunTwoActorDemo
(5 failures).

Root cause: two_actor_demo.py called as_Offer.model_validate(offer_dict) on
the trigger response. as_Offer.object_: as_Object | str | None, so
offer.object_was an as_Object, not a VulnerabilityReport. The isinstance
fallback created a NEW VulnerabilityReport with a fresh UUID which was
never stored.

Fix: Replaced the broken parse + fallback chain with
parse_submit_report_offer(offer_dict) from the factory package, which coerces
the dict to RmSubmitReportActivity and returns (report, offer) with the
correct stable IDs.
