## Issue #429 — PAD: Implement stale-claim sweeper GitHub Actions workflow

Implemented the stale-claim sweeper as a new GitHub Actions workflow
`.github/workflows/stale_claim_sweeper.yml`, satisfying PAD-10 and PAD-14-003.

The workflow runs weekly (Monday 09:00 UTC) and on `workflow_dispatch`. It
scans all `task/*` branches, checks each against orphan criteria (assigned
issue, no open PR, no branch commit within 3 days, no existing `stale-claim`
label), then applies the label and posts a comment requesting human review.
It does not close issues or delete branches.
