## IDEA-26043003 append-history should accept frontmatter fields as CLI parameters

Currently agents must write full YAML frontmatter inline in the heredoc
passed to `append-history`. This is error-prone — agents occasionally
forget required fields, indent incorrectly, or use wrong field names.

**Processed**: 2026-05-01 — combined with IDEA-26043001 and IDEA-26043002
into `specs/history-management.yaml` HM-06 and HM-07 (v1.2.0).
CLI now requires `--title` and `--source`; stdin/`--file` is body-only;
tool builds frontmatter. All skills updated.
