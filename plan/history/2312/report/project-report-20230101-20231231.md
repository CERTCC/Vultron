# Vultron Project — 2023 Work Sponsor Report

## Executive Summary

2023 was Vultron's founding year as a public open-source project, opening with repository
initialization in April and closing with two software releases: v0.5 (September) and
v0.6 (December). The year's major themes were converting the original design research
into a navigable web-based documentation site, implementing the core CVD protocol state
machinery in Python, and introducing the first behavior-tree automation that allows an
agent to simulate coordinating a vulnerability disclosure case end-to-end. A full
CI/CD pipeline, dependency hygiene, and documentation quality tooling were established
in parallel to support all future development.

---

## Project Launch and Community Infrastructure

The public repository opened in April 2023 with initial README files, user stories,
ActivityPub message examples, and a placeholder copyright notice. May brought an FAQ,
a Contributing guide, and automated Python code formatting. Formal licensing,
contribution instructions, and acknowledgments were added in July. By September, a
CODEOWNERS file and a GitHub Actions Python CI pipeline were live, providing automated
checks on every pull request from the outset.

---

## Vultron Design Documentation Goes Live

The most significant Q3 milestone was converting the *Designing Vultron* SEI technical
report into an interactive, searchable documentation website using MkDocs/Material and
Mermaid diagrams (see [PR #1](https://github.com/CERTCC/Vultron/pull/1), September
2023). More than 11,000 lines of content across 219 files were restructured so that
readers can navigate the full protocol design online rather than consuming a static PDF.
This release was tagged **v0.5 / v2023.9**.

---

## Case State Protocol Implementation

Also in September, Python code and documentation were added for the three core CVD
protocol state machines — report management, embargo management, and case-state
tracking — that together describe the full life cycle of a coordinated vulnerability
disclosure case (see [PR #8](https://github.com/CERTCC/Vultron/pull/8)). Tests and
auto-generated API reference documentation were included, making the state machinery
both executable and verifiable.

---

## Behavior Tree Automation Introduced

The standout Q4 addition was a complete behavior-tree (BT) framework and CVD-agent
simulator (see [PR #21](https://github.com/CERTCC/Vultron/pull/21), November 2023).
Behavior trees provide a structured way to model and run the decision logic an agent
follows when coordinating disclosure. The contribution includes a base BT library, a
full mapping of Vultron protocol behaviors onto that framework, unit tests,
documentation, and a command-line demo (`vultrabot`) with three simulation modes —
including one that simulates a single agent coordinating an entire CVD case with itself.
This work was released as **v0.6 / v2023.12** in December.

---

## Developer Tooling and Quality Infrastructure

Q4 also saw substantial investment in project health:

- An **Architectural Decision Records (ADR) folder** was introduced to formally
  document key design choices over time.
- **Automated dependency updates** (Dependabot) were enabled; 14 dependency-bump
  pull requests were merged cleanly before year-end.
- **Markdown linting** (markdownlint-cli2) was added with CI enforcement, keeping
  documentation consistent.
- **CI path filtering** was introduced so Python and markdown checks run only when
  relevant files change, reducing unnecessary build time.
- A **print-to-PDF plugin** was added to the documentation site.

In total, 28 pull requests were merged in 2023 — 14 substantive and 14 automated
dependency updates, all merged cleanly.
